 
## 1. COLOR SYSTEM
 
### 1.1 Brand Colors
 
| Token Name               | Hex       | Usage                                         |
|--------------------------|-----------|-----------------------------------------------|
| `color-brand-red`        | `#B7042C` | EvalRight logo — primary brand red            |
| `color-brand-purple`     | `#3B1D7D` | EvalRight logo — secondary brand purple       |
 
### 1.2 Background Colors
 
| Token Name               | Hex       | Usage                                         |
|--------------------------|-----------|-----------------------------------------------|
| `color-bg-page`          | `#F6F6F6` | Full page/body background                     |
| `color-bg-form-section`  | `#F2F2F2` | Section/group backgrounds inside forms        |
| `color-bg-billing`       | `#FFF3CF` | Billing information section background        |
| `color-bg-white`         | `#FFFFFF` | Card/form container background                |

| Token Name               | Hex       | Usage                                         |
|--------------------------|-----------|-----------------------------------------------|
| `color-text-primary`     | `#6C7589` | Default body text, labels, placeholders       |

| `color-text-link`        | `#3B1D7D` | Hyperlinks (use brand purple)                 |
| `color-text-error`       | `#B7042C` | Validation error messages (use brand red)     |

## 3. SPACING SYSTEM
 
Use an **8px base grid**.
 
| Token           | Value  | Usage                                  |
|-----------------|--------|----------------------------------------|
| `space-xs`      | 4px    | Icon gaps, tight row gaps              |
| `space-sm`      | 8px    | Input inner padding (vertical)         |
| `space-md`      | 16px   | Input inner padding (horizontal), gaps |
| `space-lg`      | 24px   | Section internal padding               |
| `space-xl`      | 32px   | Section-to-section spacing             |
| `space-2xl`     | 48px   | Page top/bottom padding                |

## 7. RESPONSIVE BREAKPOINTS
 
| Breakpoint   | Width         | Layout Change                             |
|--------------|---------------|-------------------------------------------|
| Desktop      | ≥ 1024px      | 2-column grid inside sections             |
| Tablet       | 768px–1023px  | 2-column grid, reduced padding            |
| Mobile       | < 768px       | 1-column stack, full-width inputs         |
 
- Form card: full-width on mobile, `max-width: 600px` on tablet, `max-width: 960px` on desktop.
- Padding on mobile: `16px`
- Section blocks stack vertically on mobile

## 9. VALIDATION RULES
 
| Field            | Rules                                                   |
|------------------|---------------------------------------------------------|
| Username         | Required, min 3 chars, alphanumeric + underscore only   |
| Email            | Required, valid email format                            |
| Password         | Required, min 8 chars, 1 uppercase, 1 number            |
| Confirm Password | Required, must match Password                           |
| First Name       | Required, text only                                     |
| Last Name        | Required, text only                                     |
| Address Line 1   | Required                                                |
| City             | Required                                                |
| ZIP              | Required                                                |
| Country          | Required                                                |
| Terms checkbox   | Must be checked before submit                           |
 
Error display: show error text (12px, `#B7042C`) below the field and add red border `#B7042C` on the field.
